The basics

The document follows a certain structure which must be followed:

NEVER

edit the top line of any sheet in
the doc or the tab name!
The bot checks these to
check for compatibility

[user]

when [user] is in a command
[user] will be replaced with the
username of the person who
ran the command.

ALWAYS

make sure to follow the example
commands. You have to follow
the syntax or else the bot will
not be able to run the commands!

Syntax

Commands do not have to start with '!'
They can start with anything. If a command starts
with '$' only the streamer can run it.
For example:

 Mod: !command add $!streamer You are the streamer, [user]!
 Bot: Command '!streamer' added
 Viewer: !streamer
(Notice how the bot does not respond since the viewer is not the streamer)
 Streamer: !streamer
 Bot: You are the streamer, Streamer!
If you have '&' instead of '$' only mods and the streamer can run the command
A command without either or these can be run by anyone.

One more thing...


Not all commands need to start with '!'; they can start with anything.
 Streamer: !command add Hello [user]! How are you today?
 Bot: Command 'Hello' added
 Viewer: Hello
 Bot: Hello Viewer! How are you today?
 Mod: hello
(Commands are case sensitive! You must say 'Hello' not 'hello' to run the command!)
Commands are case sensitive. In the example above they have to say 'Hello'
for the command to run. If you say 'hello' the command will NOT run.

If you want to have a separate command if the user is a mod (&)
or if the user is the streamer ($) or just a normal viewer there is a way to do this.
Here's an example of what the commands sheet would look like:
Command Command response
$!user Hello, [user]! You are the Streamer!
&!user Hello, [user]! You are a Mod!
   !user Hello, [user]! You are a Viewer!
With this layout, anyone who runs '!user' will see if they are the streamer, a mod, or just a viewer
